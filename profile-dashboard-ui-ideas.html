Profile Dashboard
Redesign Concepts

A set of UI ideas to transform the current profile page into a data-rich, developer-grade dashboard that communicates mastery at a glance — edge-to-edge, information-dense, and visually distinctive.

01

What's Wrong With the Current Design

Issue

Cards are visually identical — same size, same weight, same padding. Nothing draws the eye to what matters most (rank, score, progress).

Issue

Topic Mastery shows plain text rows. There's no visual sense of scale — 5/112 looks the same as 1/206.

Issue

The layout uses a centered, max-width card grid that leaves huge empty gutters — not edge-to-edge.

Issue

No hero moment. The user's rank (#1, Top 1%) is buried in the tiny sidebar. It should be the centrepiece.

Issue

Badges are displayed as a flat icon grid with no context — earned vs locked look nearly the same at a glance.

Issue

The heatmap shows a loading spinner. Consistency data is arguably the most motivating stat — it needs prominence.

02

Design Principles for the Redesign

🎯

Data Density Over Decoration

Every pixel should carry information. Avoid filler illustrations. Use whitespace deliberately, not generously.

⚡

Hero the Achievement

Global Rank #1 should be unmissable. Design the layout so the user's most impressive stat greets them immediately.

📐

Edge-to-Edge Structure

Use a full-bleed layout with a sidebar column and content columns — no floating card islands in empty grey space.

📊

Visual Progress, Not Just Numbers

Progress bars, sparklines, and heatmap cells give instant comprehension. Numbers alone require cognitive load.

🎨

Consistent Visual Language

Easy = green. Medium = amber. Hard = red. Use this colour grammar everywhere — tags, bars, badges, quests.

🧑‍💻

Developer Aesthetic

Lean into monospace for numbers and stats. It signals precision and appeals to a technical audience.

Dark sidebar identity panel
User avatar, rank badge (#1 in large monospace), solved/score/badge summary. Collapses to icon-only on narrow screens.
Full-width 3-column stat hero
Solved, Score, and % Complete displayed in oversized monospace with animated count-up on load.
Horizontal progress bars for topics
Each topic row shows name + filled/track bar + fraction. Bar colour encodes topic difficulty category.
Full-width GitHub-style heatmap
7 rows (Mon–Sun), 52 columns (weeks). Intensity from border-only to solid accent blue. Tooltip on hover shows date + problems solved.
Badge grid with earned/locked states
Earned badges glow with a soft drop shadow. Locked badges are desaturated with a lock icon. Hover shows unlock condition.
Stacked bar for problem distribution
Easy/Medium/Hard shown as separate bars with exact counts and a thin percentage label at the end of each fill.

XP-style hero progress bar
A full-width bar beneath the user's name showing Score/500 as an experience point bar — complete with level number and next milestone label.
Radar / spider chart for skill areas
A hexagonal radar chart mapping 6 topic areas. Immediately shows which skill is strong vs untouched. Rendered via SVG or Chart.js.
Quest progress with micro-animations
Each quest card has a thin progress bar that glows when near completion. A subtle pulse animation on active quests.
Badge wall with glow effect
Earned badges emit a subtle coloured glow (e.g. First Blood glows red). Locked badges show the unlock condition inline, not in a tooltip.
Next milestone callout
A highlighted card showing exactly what to do next: "Solve 36 more questions to unlock SQL Ninja badge." Makes the path forward obvious.
Full-width thin progress bars (terminal style)
Topic rows span the full content width. Bars are 4px tall with a retro block-fill look. Percentage and fraction shown at both ends.

Rank + XP hero band (pinned top)
A slim full-width strip below the top nav. Shows avatar, name, rank badge, and a prominent XP progress bar. Always visible — doesn't scroll away.
3-column mid row (40 / 30 / 30)
Topic Mastery takes 40% (it's the primary data). Quests take 30%. Placement Prep takes 30%. Each column scrolls independently if needed.
"Next Badge" callout inside Quests column
A highlighted mini-card that computes and shows the single closest unlockable badge — removes ambiguity about what to do next.
Full-width heatmap with summary stats
Heatmap row spans the full content width. A summary bar beneath it shows: current streak, all-time best, days active this month.
Badge row (earned/locked) with inline labels
Single horizontal row of badge icons. Earned = full colour + star. Locked = grey. Click to expand into a modal with full badge details.
Monospace numbers everywhere
All counts, percentages, and ranks use JetBrains Mono or similar. Creates a terminal/precision aesthetic that speaks to developers.

Display / UI Body
Inter
800 weight for hero numbers and headings. 400 for body text. Tight letter-spacing (−0.03em) on large sizes. Already loaded in most modern stacks via Google Fonts.
Stats / Numbers / Code
#1 · 160pts · 3%
JetBrains Mono or Fira Code for all numeric values: rank, score, question counts, percentages. Tabular figures so columns align perfectly.
08

Key Interactions to Implement

Page Load
Stat numbers animate from 0 → real value over 800ms with an easing curve. XP bar fills from left with a slight spring. Creates a satisfying "dashboard powering up" feel.
Heatmap Hover
Tooltip appears above the cell: date + "X questions solved". Cell brightens on hover. Neighbouring cells dim slightly to focus attention.
Badge Hover
Earned badges scale up 1.05x with a glow ring. Locked badges show a tooltip with the exact unlock condition (e.g. "Achieve a 3-day practice streak").
Topic Row Click
Clicking a topic in Topic Mastery navigates directly to the Practice page filtered to that topic — shortcut to continue learning from the profile.
Quest Click
Clicking a Quest opens a modal or side-panel showing all questions that count toward that quest's completion, filtered and highlighted.
Placement Prep Arrow
The arrow button on Placement Prep rows navigates directly to a random unsolved question of that difficulty/topic. Reduces friction from profile → practice.
09

Implementation Checklist for the Agent

Recommended Direction: Concept C (Analytics Pro). It retains the existing light-mode feel, is the least disruptive to the current codebase, and is the most scalable as new features are added. The dark sidebar already exists — just extend and enrich it.
Replace card grid with zone layout
Use CSS Grid with named template areas. No more floating card islands — full-bleed columns that meet the sidebar edge.
Add XP hero band
New component: slim top bar with avatar, name, rank, and a progress bar (score/500). Sticky or pinned above the main content.
Upgrade Topic Mastery rows
Add filled progress bars next to each topic. Width = solved/total × 100%. Colour encodes topic category (use existing difficulty colours).
Build real heatmap
52-week × 7-day grid. Populate from daily solved data. Use 5 intensity steps (0, 1–2, 3–5, 6–9, 10+). Add tooltip on hover.
Redesign Badge display
Earned = coloured icon + star. Locked = greyscale + lock. Add unlock condition text under each locked badge name.
Add "Next Badge" computation
Calculate the closest locked badge by progress. Show it as a highlighted card: "36 more questions to unlock SQL Ninja."
Load-in animations
Animate stat counters on mount. Stagger widget appearance with 50ms delays. XP bar fill animation. Use CSS transitions, not heavy JS libraries.
Switch stat numbers to monospace font
Add JetBrains Mono or Fira Code for all numerical values. One CSS class (.stat-num) applied globally keeps it consistent.
